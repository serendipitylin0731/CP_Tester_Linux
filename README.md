# CP Tester（奶娃版）

> version: linux-0.1.0

> Editor: serendipity_lin@sjtu.edu.cn

![alt text](https://raw.githubusercontent.com/serendipitylin0731/CP_Tester_Linux/main/media/sets/c3e14ac787413473c5e74efbd69ee80f.jpg)

一款专为算法竞赛/编程练习或出题测试设计的 VS Code 插件，提供便捷的本地测试与评测功能。

## 功能简介

CP Tester 在 VS Code 侧边栏提供一个测试面板，帮助你在提交代码前快速验证程序正确性。支持手动输入测试数据、从文件夹批量加载，或直接运行已编译好的可执行文件，自动运行并比对输出结果。

### 核心特性

- **多语言支持**：自动识别并编译运行 `.c`、`.cpp`、`.py` 文件
- **手动输入**：可手动添加最多 **5 组**测试数据（输入 + 期望输出）
- **批量导入**：从文件夹导入 `.in` / `.out`（`.ans`）数据对，最多 **100 组**
- **直接运行可执行文件**：选择已编译好的可执行文件，跳过编译直接测试
- **状态条展示**：顶部圆角矩形实时显示所有测试点评测状态（AC / WA / TLE / MLE / CE / RE / Pending）
- **结果对应显示**：每个测试点下方独立显示评测结果、运行时间和内存占用
- **可自定义限制**：支持调整时间限制（默认 1000 ms，最大 60000 ms）和内存限制（默认 256 MiB，最大 2048 MiB）
- **状态图片展示**：根据整体评测结果在面板顶部显示对应的动态图片

### 评测状态

| 状态 | 缩写 | 说明 |
|------|------|------|
| `Accepted` | AC | 答案正确 |
| `Wrong Answer` | WA | 输出与期望不匹配 |
| `Time Limit Exceeded` | TLE | 运行时间超过限制 |
| `Memory Limit Exceeded` | MLE | 内存使用超过限制 |
| `Runtime Error` | RE | 程序运行时出错 |
| `Compile Error` | CE | 编译失败 |
| `Pending` | — | 正在评测中 |


## 安装插件

### git 仓库下载

下载整个项目，在根目录运行打包指令：
```shell
npx vsce package --out build/
```

打包好的 `.vsix` 文件在 `build/` 文件夹中

### 从 VSIX 安装

1. 下载 `cp-tester-x.x.x.vsix` 文件。
2. 打开 VS Code，点击左侧活动栏的**扩展图标**（四个方块）。
3. 点击扩展面板右上角的 `...`（更多操作），选择 **从 VSIX 安装...**。
4. 选择下载的 `.vsix` 文件，等待安装完成。
5. 安装成功后，按 `Ctrl+Shift+P` 打开命令面板，输入 `Reload Window` 重新加载窗口即可使用。

## 使用方法

### 打开面板

点击 VS Code 左侧活动栏的图标打开 CP Tester 面板。

### 操作区按钮

| 按钮 | 功能 |
|------|------|
| `+` | 添加一组测试数据（输入 + 期望输出），手动模式最多 5 组 |
| `📁` | 从文件夹批量导入 `.in` / `.out`（`.ans`）测试数据，最多 100 组 |
| `📥` | 选择已编译好的可执行文件，跳过编译直接运行测试 |
| `▶ Run` | 开始评测 |
| `?` | 查看功能说明 |

### 手动输入测试

1. 在编辑器中打开需要测试的程序文件（`.c` / `.cpp` / `.py`）。
2. 在 CP Tester 面板中点击 `+` 按钮添加测试用例。
3. 在输入框中填写**输入数据**和**期望输出**。
4. （可选）在配置区域修改时间和内存限制。
5. 点击 `▶ Run` 按钮开始评测。
6. 评测时顶部状态条显示每个测试点的实时状态，每个测试点下方显示详细结果（状态、时间、内存）。

### 从文件夹加载数据

1. 准备一个文件夹，其中包含成对的测试数据文件：
   - 输入文件：`.in` 后缀
   - 输出文件：同名的 `.out` 或 `.ans` 后缀
2. 在面板中点击 `📁` 按钮，选择该文件夹。
3. 插件会自动加载所有合法的数据对并显示在面板中（超过 100 组会自动截断）。
4. 点击 `▶ Run` 即可批量评测。
5. 点击文件夹路径右侧的 `✕` 可退出文件夹模式，回到手动输入。

### 导入可执行文件直接测试

1. 点击 `📥` 按钮，选择一个已编译好的可执行文件。
2. 面板会显示当前使用的可执行文件路径。
3. 点击 `▶ Run` 直接运行该可执行文件进行测试，不再编译当前编辑器中的代码。
4. 点击可执行文件路径右侧的 `✕` 可清除，恢复为编译当前代码。

### 图片展示

面板顶部会根据当前评测结果展示对应的图片，支持以下状态文件夹：

- `sets/None/` — 未开始
- `sets/Pending/` — 评测中
- `sets/AC/` — 全部通过
- `sets/CE/` — 编译错误
- `sets/RE/` — 运行时错误
- `sets/MLE/` — 内存超限
- `sets/TLE/` — 超时
- `sets/WA/` — 答案错误

若想自定义显示图片，将 `.png` 或 `.gif` 图片放入 `media/sets/` 下对应的状态文件夹中，重新打包文件，插件会随机抽取展示。

## 配置项

在 VS Code 设置中搜索 `CP Tester` 可修改以下选项：

| 配置项 | 默认值 | 范围 | 说明 |
|--------|--------|------|------|
| `cpTester.timeLimit` | 1000 | 1 ~ 60000 | 默认时间限制（毫秒） |
| `cpTester.memoryLimit` | 256 | 1 ~ 2048 | 默认内存限制（MiB） |

## 注意事项

- 运行测试前请确保已保存当前文件。
- 时间/空间超限判断：超过设定值即判定为超限；超过设定值两倍时会被强制截停。
- 输出比对时忽略行尾空格差异。
- 导入可执行文件模式下，不再编译当前编辑器中的代码，直接运行指定的可执行文件。
